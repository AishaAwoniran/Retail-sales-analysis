# Retail-sales-analysis

## Project Overview
This project uses a fictitious retail sales dataset to demonstrate an end-to-end data analysis workflow. Leveraging Excel, SQL, and Power BI, it explores, cleans, and visualizes data to uncover insights into revenue trends, customer behavior, store performance, and key business KPIs.

## Key Questions
- What is the total revenue generated, and how does it change over time?
- How many customers contribute to overall sales?
- Which cities generate the highest revenue?
- How do individual stores perform within each city?
- Which products are the top revenue drivers?
- Are there noticeable trends or patterns in sales across different periods?
- How is revenue distributed across locations and product categories?

## Description of the Dataset
The dataset used in this project is a fictitious retail sales dataset generated by ChatGPT for analytical purposes. It contains transactional-level data capturing sales activities across multiple cities and store locations.

Key fields in the dataset include transaction identifiers, product details, store and city information, sales dates, quantities, unit prices, and total revenue. Each record represents a single sales transaction, enabling detailed analysis of revenue trends, customer activity, and product performance.

The dataset was intentionally structured to simulate real-world retail data, including common data quality issues such as missing values, duplicates, inconsistent text formats, and invalid entries, which were addressed during the data cleaning process

## Tools used
Excel: Initial data exploration and quality assessment
SQL(SSMS): Data cleaning, validation, and transformation (handling nulls, duplicates, formatting, and data integrity checks)
Power BI: Data modeling, KPI creation, and interactive dashboard visualization

    customer_id,
    COUNT(DISTINCT Record_id) AS total_orders,
    SUM(revenue) AS total_spent,
